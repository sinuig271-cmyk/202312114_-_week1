## 1. 개요 (Overview)

### 1.1 제품 목적

**Universal Approximation Theorem(UAT, 보편 근사 정리)**의 이론적 의미와 동작 원리를 직관적으로 이해할 수 있도록, 이론 설명 · 뉴런 수 증가에 따른 실시간 근사 시각화 · 실제 MLP 학습을 통한 근사 결과 확인을 하나의 데스크탑 GUI로 제공한다.

1D 함수 근사(입력 1개 → 출력 1개)를 중심으로, 뉴런 수 슬라이더 조작만으로 근사 정밀도가 향상되는 과정을 눈으로 확인할 수 있다.

### 1.2 배경 및 문제 정의

| 문제 | 설명 |
|------|------|
| 추상적 정리 | "임의의 연속 함수를 근사할 수 있다"는 명제가 직관적으로 와닿지 않는다 |
| 뉴런 수의 의미 불명확 | 은닉층 뉴런 수가 늘어나면 왜 더 잘 근사하는지 체감하기 어렵다 |
| 이론-실험 단절 | UAT 수식을 알아도 실제 MLP 학습 결과와 연결 짓기 어렵다 |
| 활성화 함수 영향 불명확 | Sigmoid vs ReLU 등 활성화 함수 선택이 근사에 어떤 영향을 주는지 모른다 |

### 1.3 제품 비전

> "슬라이더를 오른쪽으로 밀수록 구불구불한 곡선이 목표 함수에 점점 달라붙는다 — 뉴런 하나하나가 근사 조각을 맡고 있음을 눈으로 확인한다."

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 탭 1 — UAT 이론 개념 설명 (F-01)

UAT의 수학적 배경과 직관적 의미를 구조화된 텍스트, 수식, 미니 다이어그램으로 설명하는 정적 학습 콘텐츠 탭이다.

#### F-01-A. 이론 섹션 구성

| 섹션 번호 | 제목 | 주요 내용 |
|-----------|------|-----------|
| 1 | UAT란 무엇인가? | 정리의 직관적 의미, 역사적 배경 (Cybenko 1989, Hornik 1991) |
| 2 | 수학적 정의 | 정리의 공식 수식, 조건 (compact domain, continuous function) |
| 3 | 핵심 조건 | 비선형 활성화 함수의 필요성, squashing function 개념 |
| 4 | 은닉층 뉴런의 역할 | 각 뉴런이 기저 함수(basis function) 역할을 하는 직관 |
| 5 | 뉴런 수와 근사 오차 | 뉴런 수 ↑ → 근사 오차 ↓ 관계, ε-근사 개념 |
| 6 | 활성화 함수의 조건 | Sigmoid, ReLU가 UAT를 만족하는 이유 |
| 7 | UAT의 한계 | 깊이(depth) vs 너비(width), 학습 가능성과 구분 |
| 8 | 실제 의미 | MLP가 "어떤 함수든 근사할 수 있다"는 것의 실용적 의미 |

#### F-01-B. 핵심 수식 표시

- Matplotlib `mathtext` 렌더링 → PNG → `QLabel` 삽입
- 대표 수식:

```
∀ε > 0, ∃N ∈ ℕ 및 신경망 F_N 이 존재하여
sup_{x∈K} |f(x) - F_N(x)| < ε
```

#### F-01-C. 섹션 목차 사이드바

- `QListWidget` 기반 클릭 시 해당 섹션으로 스크롤 이동
- 현재 스크롤 위치에 따라 목차 항목 자동 하이라이트

### 4.2 탭 2 — 근사 시각화: 슬라이더 모드 (F-02)

학습 없이 **뉴런 수 슬라이더**만으로 실시간 근사 결과를 확인하는 인터랙티브 시각화 탭이다. 각 뉴런이 기여하는 기저 함수 조각을 함께 표시하여 UAT의 직관을 체험할 수 있도록 한다.

#### F-02-A. 목표 함수 선택

| 함수 | 수식 | 특징 |
|------|------|------|
| sin(x) | `sin(x)` | 주기성, 부드러운 곡선 |
| x² | `x²` | 단순 비선형 |
| \|x\| | `abs(x)` | 비미분점 존재 |
| 계단 함수 | `step(x)` | 불연속, UAT 한계 체험 |
| 사용자 정의 | 수식 직접 입력 | 임의 함수 실험 |

- `x` 범위: 기본 `[-π, π]`, 조절 가능

#### F-02-B. 뉴런 수 슬라이더 (핵심 기능)

- **뉴런 수 슬라이더**: 1 ~ 64 (기본값 4)
- 슬라이더 조작 시 근사 곡선이 실시간으로 업데이트 (≤ 100ms)
- 슬라이더 아래에 현재 뉴런 수 및 **근사 오차(MSE)** 수치 표시

#### F-02-C. 기저 함수 분해 시각화 (핵심 차별 기능)

UAT의 핵심 직관인 "각 뉴런이 하나의 기저 함수 조각을 담당한다"를 시각화한다.

- **메인 그래프**: 목표 함수(점선) + 근사 함수(실선) 오버레이
- **기저 함수 패널**: 각 뉴런의 개별 출력 함수 곡선을 반투명하게 오버레이
  - 뉴런 수 ≤ 16: 모든 기저 함수 개별 표시
  - 뉴런 수 > 16: 대표 10개만 표시
- **오차 영역(Error Band)**: 목표 함수와 근사 함수 사이의 차이를 음영 영역으로 표시

#### F-02-D. 활성화 함수 선택

- `QComboBox`: Sigmoid, ReLU, Tanh, GELU
- 변경 시 근사 곡선 즉시 갱신

#### F-02-E. 근사 품질 지표 패널

| 지표 | 설명 |
|------|------|
| MSE | 목표 함수와 근사 함수 간 평균 제곱 오차 |
| Max Error | 최대 절대 오차 |
| 근사 품질 등급 | MSE 기준 Poor / Fair / Good / Excellent 4단계 |

### 4.3 탭 3 — MLP 학습을 통한 근사 (F-03)

실제 MLP를 NumPy로 학습시켜 목표 함수를 근사하는 과정을 시각화하는 탭이다. 슬라이더 모드(탭 2)와 달리 **실제 Gradient Descent**로 가중치를 업데이트하며 근사가 개선되는 과정을 확인한다.

#### F-03-A. 하이퍼파라미터 설정

| 파라미터 | 입력 방식 | 범위 |
|----------|-----------|------|
| 은닉 뉴런 수 | QSpinBox + 슬라이더 | 1 ~ 64 |
| 학습률 | QDoubleSpinBox | 0.0001 ~ 1.0 |
| Epochs | QSpinBox | 100 ~ 50,000 |
| 활성화 함수 | QComboBox | Sigmoid, ReLU, Tanh |
| 가중치 초기화 | QComboBox | Random, Xavier, He |
| 목표 함수 | QComboBox | 탭 2와 동일 5종 |

#### F-03-B. 실시간 학습 시각화

- **근사 그래프**: 학습 진행에 따라 근사 곡선이 목표 함수에 수렴하는 과정을 실시간 표시
- **Loss 곡선**: Epoch별 MSE Loss 실시간 플로팅
- **현재 Epoch / Loss 수치** 표시

#### F-03-C. 학습 제어

| 버튼 | 동작 |
|------|------|
| ▶ 학습 시작 | MLP 학습 시작 |
| ⏸ 일시 정지 | 현재 epoch에서 정지 |
| ⏹ 초기화 | 가중치 재초기화, 그래프 초기화 |
| ⏩ 빠른 학습 | 전체 epoch 즉시 완료 후 결과만 표시 |

#### F-03-D. 탭 2 vs 탭 3 비교 관점 안내

- 학습 완료 후 안내 텍스트: "슬라이더 모드(탭 2)는 이상적인 가중치를 가정한 수학적 근사이며, 학습 모드(탭 3)는 실제 경사 하강법으로 수렴하는 과정입니다."

## 8. 근사 방식 명세 (탭 2 슬라이더 모드)

탭 2의 슬라이더 모드는 학습 없이 **해석적으로 계산된 이상적 가중치**를 사용한다.

- **방식**: 단일 은닉층 MLP의 이상적 근사를 구현하기 위해 `numpy.linalg.lstsq`로 출력층 가중치를 최소 제곱 풀이
- **은닉층**: Sigmoid(또는 선택된 활성화 함수) 적용, 입력 가중치는 균등 분포로 고정 배치
- 뉴런 수 변경 시 은닉층 재구성 후 최소 제곱 풀이 반복 → 실시간 갱신

## 9. 제약사항 및 가정

- 시각화는 **1D 함수 근사**에 집중 (2D 곡면 근사는 v2.0 고려)
- 사용자 정의 수식은 Python `eval()` 샌드박스 처리, 위험 키워드 필터링
- 계단 함수는 엄밀한 UAT 조건(연속 함수)을 위반하므로, 탭 1 섹션 7에서 한계 사례로 함께 설명
- MLP 연산은 NumPy만 사용 (PyTorch/TensorFlow 미사용)

## 1. 개요 (Overview)

### 1.1 제품 목적

딥러닝의 핵심 동작 원리인 **Forward Propagation(순전파)**을 직관적으로 이해할 수 있도록, 네트워크 구조 시각화 · 가중치 표현 · 활성화값 표시 · 단계별 애니메이션을 제공하는 데스크탑 GUI 애플리케이션이다.

머신러닝 입문자부터 실무자까지 **범용 사용자**를 대상으로 하며, Fully Connected 네트워크 및 CNN 구조를 모두 지원한다.

### 1.2 배경 및 문제 정의

| 문제 | 설명 |
|------|------|
| 추상적 계산 흐름 | Forward Propagation의 데이터 흐름이 코드나 수식만으로는 직관적으로 파악하기 어렵다 |
| 가중치·활성화값 연결 부족 | 각 뉴런의 가중치 합산 → 활성화 함수 → 출력값으로 이어지는 과정을 시각적으로 경험하기 힘들다 |
| CNN 내부 구조 불명확 | 합성곱·풀링 연산이 실제로 특징 맵에 어떤 영향을 주는지 체감하기 어렵다 |
| 정적 자료의 한계 | 교재나 강의 슬라이드의 정적 다이어그램으로는 "흐름"을 체험할 수 없다 |

### 1.3 제품 비전

> "앞으로 버튼을 누를 때마다 데이터가 한 레이어씩 살아서 움직인다 — 가중치, 활성화값, 연산 과정을 한 화면에서 단계별로."

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 네트워크 구조 설정 (F-01)

#### F-01-A. Fully Connected (Dense) 네트워크 구성

- 레이어 수(1 ~ 6), 각 레이어별 뉴런 수(1 ~ 16) 를 사용자가 직접 설정한다
- 입력 레이어 뉴런 수와 입력값을 직접 입력할 수 있다
- 각 레이어에 적용할 Activation Function을 개별 선택한다 (ReLU, Sigmoid, Tanh, Linear)
- 가중치(Weight) 및 편향(Bias) 초기화 방식을 선택한다 (Random, Xavier, He, 직접 입력)
- 설정 변경 시 네트워크 다이어그램이 즉시 갱신된다

#### F-01-B. CNN 네트워크 구성

- Conv2D 레이어: 필터 수, 커널 크기(3×3, 5×5), Stride, Padding 설정
- Pooling 레이어: Max Pooling / Average Pooling, 풀 크기 선택
- Flatten + Fully Connected 레이어 연결 지원
- 입력 이미지: 샘플 이미지 선택 또는 사용자 이미지 업로드 (8×8 ~ 32×32 권장)

#### F-01-C. 프리셋 네트워크

빠른 탐색을 위한 사전 정의 네트워크를 제공한다.

| 프리셋 이름 | 구조 | 용도 |
|-------------|------|------|
| XOR Solver | 2→4→1 FC | 이진 분류 입문 |
| Simple Classifier | 4→8→4→3 FC | 다중 분류 입문 |
| Mini CNN | Conv→Pool→FC | CNN 기초 이해 |

---

### 3.2 네트워크 시각화 (F-02)

#### F-02-A. FC 네트워크 다이어그램

- 뉴런을 원(Circle)으로, 레이어 간 연결을 선(Edge)으로 표현한다
- 가중치 값에 따라 선의 **굵기** 및 **색상**을 다르게 표시한다
  - 양수 가중치: 파란 계열, 음수 가중치: 빨간 계열
  - 절댓값이 클수록 선이 굵어진다
- 각 뉴런 안에 현재 **활성화값**을 숫자로 표시한다
- 활성화값 크기에 따라 뉴런의 **배경 색상 밝기**가 변한다 (값이 클수록 밝음)

#### F-02-B. CNN 특징 맵 시각화

- 각 Conv 레이어의 출력 특징 맵(Feature Map)을 그리드 형태로 표시한다
- 커널(필터)이 입력 위를 슬라이딩하는 과정을 애니메이션으로 표현한다
- Pooling 레이어 적용 전후 특징 맵 크기 변화를 나란히 표시한다

---

### 3.3 단계별 애니메이션 (F-03)

#### F-03-A. Step-by-step 진행 방식

- **[▶ 다음 단계]** 버튼을 클릭할 때마다 계산이 한 레이어씩 진행된다
- 각 스텝에서 아래 내용이 순서대로 하이라이트된다:
  1. 현재 처리 중인 레이어의 뉴런과 연결선이 강조된다
  2. 가중치 합산(z = Wx + b) 계산 결과가 해당 뉴런 위에 표시된다
  3. 활성화 함수 적용 후 출력값(a = f(z))이 뉴런 내부에 반영된다
- **[◀ 이전 단계]** 버튼으로 이전 스텝으로 되돌아갈 수 있다
- **[↺ 초기화]** 버튼으로 초기 상태로 리셋한다
- 현재 스텝 번호와 전체 스텝 수를 진행 바(Progress Bar)로 표시한다

#### F-03-B. 자동 재생 모드 (선택)

- **[▷ 자동 재생]** 버튼으로 일정 속도로 자동 진행한다
- 재생 속도 슬라이더(0.5× ~ 3×)를 제공한다

---

### 3.4 계산 상세 패널 (F-04)

- 현재 스텝의 계산 과정을 수식으로 표시한다
  - 예: `z = W·x + b = [0.3×1.0 + 0.7×0.5 + ...] + 0.1 = 1.25`
  - 예: `a = ReLU(1.25) = 1.25`
- 이전 스텝의 계산 이력을 스크롤 가능한 로그 형태로 기록한다
- 행렬 표현 탭: 전체 레이어의 가중치 행렬(W)과 입력 벡터(x)를 테이블로 표시한다

---

### 3.5 입력값 설정 (F-05)

- FC 네트워크: 입력 뉴런별 수치 값을 스핀박스로 직접 입력한다
- CNN: 샘플 이미지 갤러리에서 선택하거나 외부 이미지를 업로드한다
- **[랜덤 입력]** 버튼으로 무작위 입력값을 자동 생성한다

---

### 3.6 한/영 언어 전환 (F-06)

- 상단 툴바에 언어 전환 버튼(KO / EN)을 제공한다
- 전환 시 모든 UI 레이블, 설명 텍스트, 수식 설명이 즉시 변경된다
- 수식 자체(수학 기호, 숫자)는 언어와 무관하게 동일하게 유지된다

## 5. UI/UX 요구사항

### 5.1 레이아웃 구조

```
┌──────────────────────────────────────────────────────────────────┐
│  [툴바]  앱 제목    [모드: FC | CNN]    프리셋 ▼    KO / EN       │
├─────────────────┬────────────────────────┬────────────────────────┤
│  [설정 패널]     │  [네트워크 다이어그램]   │  [계산 상세 패널]       │
│                 │                        │                        │
│  레이어 수       │   ○ ─── ○ ─── ○       │  현재 스텝 수식          │
│  뉴런 수         │   ○ ─── ○ ─── ○       │  z = Wx + b = ...      │
│  Activation     │   ○ ─── ○ ─── ○       │  a = f(z) = ...        │
│  Weight 초기화   │                        │                        │
│                 │                        │  ─────────────         │
│  [입력값 설정]   │                        │  [계산 로그]            │
│  x1: [1.0]      │                        │  Step 1: Layer 1 ...   │
│  x2: [0.5]      │                        │  Step 2: Layer 2 ...   │
│  [랜덤]          │                        │                        │
│                 ├────────────────────────┤                        │
│                 │ [◀ 이전] [▶ 다음] [↺]  │                        │
│                 │ Progress ████░░░░ 2/5  │                        │
└─────────────────┴────────────────────────┴────────────────────────┘
```

### 5.2 UX 원칙

- **단계적 공개**: 각 스텝마다 하나의 레이어만 강조하여 인지 과부하를 방지한다
- **색상 일관성**: 양수/음수 가중치 색상, 활성화값 색상 규칙을 전체 UI에서 통일한다
- **즉각적 피드백**: 네트워크 구성 변경 시 다이어그램이 즉시 갱신된다
- **범용성**: 입문자를 위한 설명 툴팁과 고급 사용자를 위한 행렬 뷰를 모두 제공한다

### 5.3 색상 규칙

| 대상 | 색상 |
|------|------|
| 양수 가중치 선 | 파란 계열 (#2563EB) |
| 음수 가중치 선 | 빨간 계열 (#DC2626) |
| 현재 활성 레이어 강조 | 노란 계열 (#FBBF24) |

| 활성화값 낮음 / 0 | 회색 (#9CA3AF) |

## 6. 지원 네트워크 구조 상세

### 6.1 Fully Connected

| 항목 | 범위 / 선택지 |
|------|---------------|
| 레이어 수 (입력 제외) | 1 ~ 6 |
| 레이어당 뉴런 수 | 1 ~ 16 |
| Activation Function | ReLU, Sigmoid, Tanh, Linear, Softmax (출력층) |

### 6.2 CNN

| 항목 | 범위 / 선택지 |

| Stride | 1, 2 |
| Padding | Same, Valid |
| Pooling | Max, Average / 풀 크기 2×2 |
| 입력 이미지 크기 | 8×8 ~ 32×32 |

## 7. 제약사항 및 가정

- 이 애플리케이션은 **교육 목적의 시각화 도구**이며, 실제 학습(Backpropagation) 기능은 v1.0 범위 외
- CNN 시각화는 2D(단채널 또는 3채널 입력)로 제한하며, 3D/배치 처리는 미지원
- 뉴런 수가 많을 경우(레이어당 > 8) 선의 일부를 생략하거나 투명도 조절로 가독성을 유지한다
- 가중치 직접 입력 모드에서 비정상 값(NaN, Inf) 입력 시 경고 후 기본값으로 대체한다
